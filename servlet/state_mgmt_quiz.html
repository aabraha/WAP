Servlet Intro Quiz

  1. Can post requests be bookmarked? What are the problems?

    Post requests can't be bookmarked in full way just like Get requests. Get requests can be bookmarked because form data are sent by appending it to url and bookmarks store those parameters as part of the url. But form data submitted using the Post method aren't sent in the URL but as body of the request.

  2. What is the purpose of request dispatching?

    Servlets may internally pass ("forward") the request processing to another local resource. RequestDispatcher is therefore used in such cases(request dispatching) to forward to another servlet or jsp in the same website as follows. 

    RequestDispatcher requestDispatcher = request.getRequestDispatcher("absolute or relative uri");
            requestDispatcher.forward(request,response);

  3. What is the difference between redirect and request dispatch?

    Redirect is sent to the client as a response while the request dispatch is performed completely in the server and the result of the forward from the request dispatch is returned as a response to the client.

  4. What is an attribute ?

    Attributes are name(String) and value(objects) pairs in the server bound to either of the three servlet API objects that is HttpServletRequest, HttpSession, ServletContext. Only request attributes are thread safe.

  5. What is the difference between attributes and parameters?

    Parameters are form data sent as part of http requests in the form of key value pairs while attributes are name(String) and value(objects) pairs bound to either of the three servlet API objects that is HttpServletRequest, HttpSession, ServletContext.

  6. What are dangers of using attributes?

    Since session level and application level attributes are shared by multiple servlets and requests they aren't thread safe.

  7. What does it mean to say that http is stateless? Give an example of a stateful protocol.

    By http is a stateless protocol, we mean after handling request and response between client and server no memory is stored about who sent and other informations. FTP is an example of stateful protocol because it store who is accessing or downloading files.

  8. Give 5 different methods for maintaining state information (count each attribute scope as one method).

    • Request Scope which is destroyed after after servlet finishes processing it.
    • Session Scope which is destroyed when user closes browser.
    • Application Scope which is destroyed when container stops.
    • Cookies which in return can be temporary and get deleted when a browser closes or permanent which gets deleted when a given maximum timeout reaches.
    • Hidden form fields

  9. How long does a session last?

    Typically a session lasts 30 minutes of inactivity but this can be adjusted so that it lasts from few seconds to a few hours.

  10. What is a cookie, and how long does a cookie last?

    A cookie is a small information sent from websites stored in the browser to keep stateful information and record users' browsing information. Cookies last in users computers until the user closes the browser.

  11. What is the purpose of URL rewriting?

    URL rewriting is another technique of session management when browser doesn't support cookies.

  12. Why does the request attribute report ‘null’ for the maintaining state demo?

    Because states weren't store at the time of calling the states.
